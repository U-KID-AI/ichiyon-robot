# いちよんロボ ver2.0 機能別設計

## 方針

ver2.0 では、機能をサーバーごとに管理します。
すべての運用データは基本的に `guild_id` を持ち、サーバーごとに分離します。

## 名言

サーバーごとに別管理します。
文字、画像、文字+画像に対応します。

主な項目:

* 本文
* 画像
* 有効/無効

## おみくじ

サーバーごとに別管理します。

主な項目:

* 結果名
* 本文
* 画像
* 出やすさ
* 有効/無効

管理画面では `weight` や `重み` ではなく「出やすさ」と表示します。

## 自動反応

サーバーごとに別管理します。

主な項目:

* トリガー
* 返答
* 画像
* 絵文字内部表記
* 一致方式
* 優先度
* 有効/無効
* 特殊効果タグ

既に自動反応には優先度を追加済みです。
複数トリガーが同時一致した場合は、優先度が高いものを優先し、同率ならランダムにします。

## NGワード

サーバーごとに別管理します。
NGワードを含む投稿には通常反応しません。

主な項目:

* NGワード
* 有効/無効

## 自動投稿

6/30自動投稿は単独機能にしません。
「自動投稿」という機能の中の1レコードとして扱います。

主な項目:

* 投稿名
* 本文
* 画像
* 投稿チャンネル
* 日付/スケジュール
* 繰り返し設定
* 有効/無効
* 投稿済み管理

6/30は以下のような設定値です。

```txt
名前: 6/30 サ終やめませんか？
日付: 毎年6月30日
本文: サ終やめませんか？
```

## 特殊効果タグ

自動反応に付与する特殊効果タグを作成・編集する機能です。
自動反応画面では、作成済みの特殊効果タグを選択して付与するだけにします。
タグ自体の作成は別機能です。

特殊効果タグ例:

* ライオ9倍
* さくらんぼ2回
* しこっち特殊抽選
* 成田カウント加算

効果タイプ候補:

* 特殊行動確率倍率
* 次回行動回数追加
* カウント加算
* モード抽選
* 疑似オフライン抽選

## デッキ検索

ver2.0の目玉機能の一つです。

これはBotにデッキを保存する機能ではありません。
Xから条件に合う最新ポストを検索して貼る機能です。

コマンド例:

```txt
@いちよんロボ デッキ エルフ
@いちよんロボ デッキ エルフ ローテーション
```

処理イメージ:

```txt
Xから該当ポスト検索
↓
BEYOND到達っぽい投稿を候補にする
↓
添付画像にQRコードがあるか確認
↓
画像内のクラスアイコンが指定クラスか判定
↓
条件に合うポストを2件程度返す
```

管理画面で設定するもの:

* 使用可能チャンネル
* 返却件数
* フォーマット未指定時の扱い
* 許可外チャンネル時の案内

デッキ一覧CRUDは不要です。

## 破壊機能

危険機能なので後回しです。
ver2.0で必ず実装する必要はありません。

実装する場合は以下を必須にします。

* ラボ限定
* 管理者ON
* 安全ガード
