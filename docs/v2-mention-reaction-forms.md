# ver2.0 メンション反応作成編集フォーム

## 目的

ver2.0管理画面で、DB上の `mention_reactions` と `mention_reaction_choices` を作成・編集するための土台です。

このPhaseでは既存Bot挙動、既存JSON処理、旧JSON編集画面は変更しません。DB管理画面から作成・編集した内容をBot実行時に反映する作業は後続Phaseで行います。

## ランダム抽選の作成

`/guilds/{guild_id}/mention-reactions/new` から新規作成できるのはランダム抽選のみです。

- `reaction_kind` はDB上では `random` として保存し、管理画面では `random_draw` と表示します。
- `is_system` は新規作成分では `false` です。
- `is_deletable` は新規作成分では `true` です。
- 検索型 `search` はシステム固定機能のため、管理画面から新規作成しません。

入力項目は、反応名、説明、有効/無効、管理者限定、キーワード/パターン、一致方式です。一致方式は `exact` / `prefix` / `regex` をフォームで選択できます。

## 編集方針

通常のランダム抽選は `editor` 以上が編集できます。

`admin_only=true` の反応と `is_system=true` の反応は重要項目として扱い、`guild_admin` 以上だけが編集できます。`viewer` は閲覧のみです。

検索型 `search` は詳細画面で仮表示しますが、検索ロジックや検索設定はこのPhaseでは編集しません。

## キーワード重複禁止

同じ `guild_id` 内で同じ `keyword` は保存できません。新規作成時は既存の `mention_reactions` 全体と照合し、編集時は自分自身を除外して重複チェックします。

重複時は「このキーワードは既存のメンション反応で使用されています。」を表示します。

`B` と `ABC` のような複数一致時は、後続のBot反映Phaseで長いキーワードを優先し、それでも同じなら作成が古いものを優先します。

## 抽選候補

ランダム抽選の編集画面では `mention_reaction_choices` を一覧表示し、候補の追加・編集ができます。

候補の項目は以下です。

- 候補名
- 本文
- 画像パス
- 出やすさ
- 有効/無効

画像アップロード本体はこのPhaseでは実装せず、`image_path` の文字列入力だけを扱います。

本文と画像パスの両方が空の場合は保存できません。出やすさは1以上の整数です。

## 削除

削除はこのPhaseでは未実装です。`is_system=true` の反応は削除不可であることを画面に明記します。新規追加分の削除は後続Phaseで扱います。
