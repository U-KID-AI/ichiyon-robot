# メンション反応の限定機能

## 方針

破壊機能は独立機能として扱わない。
管理画面の機能一覧からも外す。

今後の破壊は特殊効果タグの一種として扱う。
今回の実装では危険処理は動かさず、プリセットタグ「破壊」を作れる枠だけ用意する。

## メンション反応カテゴリ

メンション反応は3種類に分ける。

- ランダム抽選
- 検索
- 限定機能

デッキ検索は引き続き「検索」の中で扱う。
実際のデッキ検索実行は後続。

## 限定機能

限定機能は、特定ユーザーからBotへメンションされた時に、指定した特殊効果タグを必ず実行対象へ追加する仕組み。

例:

- 対象ユーザーID: ユーザーA
- 付与する特殊効果タグ: ミニいちよん

ユーザーAが `@いちよんロボ お前もしこっちよな？` と送ると、通常のメンション反応に加えてミニいちよん効果も実行対象になる。

## DB

`mention_limited_effects` を追加する。

- `guild_id`
- `discord_user_id`
- `display_name`
- `effect_tag_id`
- `description`
- `enabled`

同じ `guild_id + discord_user_id + effect_tag_id` は重複不可。

## 管理画面

場所:

`メンション反応 > 限定機能`

管理項目:

- 対象ユーザーID
- 表示名/メモ
- 付与する特殊効果タグ
- 有効/無効
- 説明
- 削除

付与できるタグは、メンション反応向けの特殊効果タグを基本にする。
管理者限定タグはサーバー管理者以上だけ扱う。

## Bot実行

DB backend時だけ有効。

メンション反応時の流れ:

1. `guild_id` と送信者のDiscordユーザーIDを見る
2. 有効な限定機能タグを取得
3. 通常の抽選候補に付与されたタグと合流
4. 重複タグを除いて実行
5. 失敗してもBot全体は止めない

検索型メンション反応でも、マッチした固定機能に対して限定機能タグがある場合は実行対象にする。
検索本体の実行は後続。
