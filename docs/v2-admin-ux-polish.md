# ver2.0 管理画面UX改善

## 目的

ver2.0管理画面を、非エンジニアの日本人ユーザーが通常運用で迷わない形に整理する。

管理画面はDB運用を前提にしつつ、Bot実行側の `ICHIYON_DATA_BACKEND=json` 互換は残す。

## 主な変更

- メンション反応は、最初に「ランダム抽選」「検索」を選ぶ画面を表示する。
- `random_draw` や `match_type` などの内部名は、通常画面では日本語ラベルで表示する。
- `integration_check_*` や `000 integration ...` の検証用データは通常一覧から隠し、「検証用データも表示」で確認する。
- 画像はパス手入力ではなく、ファイル選択でアップロードする。
- アップロード画像は `assets/images/...` 配下に保存し、DBには相対パスを保存する。
- 自動反応の「絵文字内部表記」は通常フォームから隠す。DBと実行処理は互換用に残す。
- 固定機能以外は削除できる。固定機能は日本語の理由付きで削除不可にする。

## 画像アップロード対象

- メンション反応の抽選候補画像
- 自動反応画像
- モード時アイコン
- モード突入/終了GIF
- モード返答候補画像
- 自動投稿画像

## ミニいちよん変換

追加投稿テンプレートで次の変換指定を使える。

```text
{match_1:hankaku}
```

変換内容:

- ひらがなをカタカナへ変換
- 全角カタカナを半角カタカナへ変換

例:

```text
しこっち -> ｼｺｯﾁ
```

既存の `{match_1}` はそのまま動く。未知の変換指定や存在しない変数は、Bot全体を止めずに元の表記を残す。

## 特殊効果タグのテンプレート

通常画面で案内する変数:

- `{match_1}`
- `{match_2}`
- `{user_name}`
- `{user_mention}`
- `{match_1:hankaku}`

## 検証用データ

統合確認スクリプトが作る `integration_check_*` 系データは、通常運用では一覧に表示しない。

検証時だけ各一覧の「検証用データも表示」をONにする。

## 注意

- JSONファイルや画像ファイルは削除しない。移行元バックアップとして残す。
- 本番切り替え前は、`migrate_json_to_db.py`、`seed_v2_presets.py`、`check_v2_db_integration.py` の順で確認する。
- 正規表現は上級者向けとして扱い、通常ユーザーには完全一致・前方一致・含むを優先して案内する。
