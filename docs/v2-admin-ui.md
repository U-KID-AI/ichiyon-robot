# いちよんロボ ver2.0 管理画面設計

## 目的

ver2.0 の管理画面では、Discordログイン後に権限のあるサーバーだけを表示し、サーバー別に機能を管理できるようにします。

画面遷移は以下です。

```txt
Discordログイン
↓
サーバー選択
↓
機能一覧
↓
各機能編集画面
```

## ログイン

Discord OAuth2 を使います。
独自のID/パスワードは持ちません。

ログイン後は DiscordユーザーID をもとに権限判定します。

## 権限

権限候補:

* `global_admin`: 全サーバー管理可能
* `guild_admin`: 自サーバー管理可能
* `editor`: 許可されたサーバーの通常データ編集可能
* `viewer`: 閲覧のみ

危険設定は管理者限定にします。

危険設定例:

* 機能ON/OFF
* モード有効化
* 特殊効果タグ作成/編集
* 破壊設定
* 権限設定

## サーバー選択

ログインユーザーが権限を持つサーバーだけを表示します。

表示候補:

* いちよんラボ
* ランセ地方
* レストラン

サーバーを選択すると、そのサーバーの機能一覧へ遷移します。

## 機能一覧

サーバーを選択すると、そのサーバーの機能一覧を表示します。

各行には ON/OFF スイッチを表示します。
ON/OFF部分を押すと機能の有効/無効を切り替えます。
ON/OFF以外の行部分を押すと、その機能の編集画面へ遷移します。

機能一覧の候補:

* メンション反応
* 自動反応
* NGワード
* モード
* 自動投稿
* 特殊効果タグ
* 破壊

デッキ検索は最初の機能一覧には表示しません。
デッキ検索は「メンション反応 > 検索 > デッキ検索」に配置します。

`viewer` は閲覧のみです。
`editor` は通常データを編集できますが、危険設定は変更できません。

## メンション反応管理画面

メンション反応は、Botへのメンションを起点に返答する機能です。
新規作成時は、まず「ランダム抽選」か「検索」を選びます。

### ランダム抽選

ランダム抽選は、管理画面から新規作成、編集、削除できます。
名言、おみくじ、お前も〇〇よな？はランダム抽選で実現します。

Discordでは `@Bot おみくじ` のほか、`@Bot おみくじ 10連` や `@Bot おみくじ10連` のように1～100連の連続抽選を指定できます。
回数指定がない場合は従来どおり1回だけ抽選します。

名言のメンション反応枠は削除不可にします。

同じサーバー内で既存と同じキーワードは保存不可です。
B と ABC のようなキーワードがある場合、ABC入力時はABCを優先します。
複数一致時は長いキーワードを優先し、それでも同じなら作成が古いものを優先します。

ランダム抽選の主な項目:

* キーワード
* 表示名
* 有効/無効
* 抽選候補

抽選候補の主な項目:

* 本文
* 画像
* 出やすさ
* 有効/無効
* 特殊効果タグ

### 検索

検索はシステム固定機能のみです。
管理画面から新規追加はできません。

デッキ検索は検索型で実現します。
配置は「メンション反応 > 検索 > デッキ検索」です。

## 名言編集画面

名言は単独機能ではなく、メンション反応のランダム抽選枠として管理します。

入力項目:

* 本文
* 画像
* 有効/無効

文字のみ、画像のみ、文字+画像に対応します。

## おみくじ編集画面

おみくじは単独機能ではなく、メンション反応のランダム抽選枠として管理します。

入力項目:

* 結果名
* 本文
* 画像
* 出やすさ
* 有効/無効

管理画面上の表示は `weight` や `重み` ではなく「出やすさ」にします。

## 自動反応編集画面

サーバーごとの自動反応を管理します。

入力項目:

* トリガー
* 返答
* 画像
* 絵文字内部表記
* 一致方式
* 優先度
* 有効/無効
* 特殊効果タグ

特殊効果タグは、この画面で新規作成せず、作成済みのタグを選択して付与するだけにします。

複数トリガーが同時一致した場合は、優先度が高いものを優先します。
同率ならランダムにします。

## NGワード編集画面

サーバーごとのNGワードを管理します。

入力項目:

* NGワード
* 有効/無効

NGワードを含む投稿には通常反応しません。

## モード管理画面

はゆすモード、成田モード、しこっちモードなどを個別機能にせず、「モード」という一つの機能にまとめます。

機能一覧では「モード」をON/OFFできます。
モード機能を押すとモード一覧を表示します。
モード一覧で各モードを有効/無効にできます。
さらに各モードを押すと、そのモードの設定画面へ遷移します。

各モード共通設定:

* 発動条件
* クールタイム
* モード中挙動
* 返答候補
* 終了条件
* 通知
* 見た目
* 継続時間
* 突入通知チャンネル
* 終了通知チャンネル
* 反応チャンネル
* 非反応チャンネル

反応チャンネルが未設定なら全チャンネルで反応します。
非反応チャンネルが設定されている場合、そのチャンネルでは反応しません。
優先順位は「非反応チャンネル > 反応チャンネル > 全チャンネル」です。

## 自動投稿編集画面

6/30自動投稿は単独機能にせず、「自動投稿」という機能の中の1レコードとして扱います。

入力項目:

* 投稿名
* 本文
* 画像
* 投稿チャンネル
* 日付/スケジュール
* 繰り返し設定
* 有効/無効
* 投稿済み管理

6/30の設定例:

```txt
名前: 6/30 サ終やめませんか？
日付: 毎年6月30日
本文: サ終やめませんか？
```

## 特殊効果タグ編集画面

メンション反応の抽選候補、自動反応、NGワードに付与する特殊効果タグを作成・編集します。
メンション反応本体とモードには付与しません。

入力項目:

* タグ名
* タグ色
* 管理者限定
* 効果タイプ
* 効果設定
* 追加投稿テキスト
* 有効/無効

効果設定はJSONB想定で柔軟に扱います。
追加投稿テキストは空白可です。
危険度は持ちません。

特殊効果タグ例:

* ライオ9倍
* さくらんぼ2回
* ミニいちよん
* しこっち特殊抽選
* 成田カウント加算

効果タイプ候補:

* 特殊行動確率倍率
* 次回行動回数追加
* カウント加算
* モード抽選
* 疑似オフライン抽選

## デッキ検索設定画面

Botにデッキを保存する画面ではありません。
Xから条件に合う最新ポストを検索して貼るための設定を管理します。

入力項目:

* 使用可能チャンネル
* 返却件数
* フォーマット未指定時の扱い
* 許可外チャンネル時の案内

デッキ一覧CRUDは不要です。

## 検索/絞り込み

以下の画面では検索/絞り込みを提供します。

* メンション反応
* 抽選候補
* 自動反応
* NGワード
* 特殊効果タグ
* モード
* モード返答候補
* 自動投稿

Info表示は Infoボタン + モーダルで行います。
ホバー依存にはしません。
PC/スマホ両対応を前提にします。

## 破壊機能設定画面

危険機能なので後回しです。
ver2.0で必ず実装する必要はありません。

実装する場合は以下を必須にします。

* ラボ限定
* 管理者ON
* 安全ガード
