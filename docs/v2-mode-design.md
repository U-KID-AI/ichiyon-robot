# いちよんロボ ver2.0 モード設計

## 方針

はゆすモード、成田モード、しこっちモードなどを個別機能にせず、「モード」という一つの機能にまとめます。
モードはタイプ固定ではなく、発動条件、クールタイム、モード中挙動、返答候補、終了条件、通知、見た目、チャンネル設定で構成します。

機能一覧では「モード」をON/OFFできます。
モード機能を押すとモード一覧を表示します。
モード一覧で各モードを有効/無効にできます。
さらに各モードを押すと、そのモードの設定画面へ遷移します。

## 共通設定

各モードは以下の設定を持ちます。

* モード名
* 発動条件
* クールタイム
* モード中挙動
* 返答候補
* 終了条件
* 通知
* 見た目
* チャンネル設定
* 継続時間

モード基本情報に優先度は不要です。
モード時表示名はモード名と同じなので個別設定不要です。
通常時表示名と通常時アイコンはBot固定値へ戻すため、個別設定不要です。

モード中は他の機能を一切使いません。
また、モード中は他のモードに遷移しない仕様にします。

## モード中挙動

モード中挙動は排他式です。
以下のどちらか1つだけを選びます。

* 返答モード
* オフラインモード

はゆすと成田は返答モードです。
しこっちはオフラインモードです。

返答モードでは、登録された返答候補からランダムに返します。
はゆすは返答候補を1つ登録すればよいです。

オフラインモードでは、Botプロセスを停止せず、一定時間反応しない疑似オフライン状態にします。

## 通知設定

通知設定では以下を設定できます。

* 突入時メッセージ
* 終了時メッセージ
* 突入時GIF
* 終了時GIF

「まずは女子供から殺す」は終了メッセージではなく、通常モード復帰時の固定プリセットメッセージです。
管理画面では編集しません。

## 見た目設定

見た目設定ではモード時アイコン画像を設定できます。
モード時表示名はモード名を使うため、個別設定しません。
通常時表示名と通常時アイコンはBot固定値へ戻します。

## チャンネル設定

チャンネル設定では以下を設定できます。

* 突入通知チャンネル
* 終了通知チャンネル
* 反応チャンネル
* 非反応チャンネル

反応チャンネルが未設定なら全チャンネルで反応します。
非反応チャンネルが設定されている場合、そのチャンネルでは反応しません。

優先順位は以下です。

```txt
非反応チャンネル > 反応チャンネル > 全チャンネル
```

## 発動方式

発動条件では以下を扱います。

* 確率抽選型
* カウント到達型
* 手動/テスト型

カウント到達条件では、既存カウントを使うだけでなく、新規カウントを自動作成できます。
新規カウントには以下を設定できます。

* カウント名
* カウントキー
* 説明
* 初期値
* リセット方式

## はゆすモード

確率抽選型です。
返答モードです。
返答候補からランダムに返します。
返答候補は1つ登録すればよいです。

設定項目:

* 発生確率
* クールタイム
* 継続時間
* 返答候補
* 通知設定
* 見た目設定
* チャンネル設定
* 有効/無効

現在の基本値:

* 発生確率: 1/112
* 継続時間: 180秒

クールタイム:

* 期間内1回
* 月次
* 月初リセット

## 成田モード

カウント到達型です。
返答モードです。
返答候補からランダムに返します。

対象ワード:

* お金
* データ

対象ワードを検知するたびに `:narita:` リアクションを付け、成田カウントを増やします。
22回貯まると成田モードに突入します。

クールタイム:

* 期間内1回
* 月次
* 毎月22日リセット

毎月22日に集計期間を更新し、以下をリセットします。

* 成田カウント
* 期間内発動済み状態

1期間に1回だけ成田モードになれる想定です。

成田モード中の返答候補:

* お金の代わりにデータを持つ時代が到来する
* 稼ぐより踊れ
* まねきねこアルゴリズム
* 泥だんご
* アートークン

終了時は「寿命と預金を減らしながら元のモードに戻る」。

## しこっちモード

通常反応として「しこっち」に「しこっちきたぁぁぁ」と返します。

1/444で「しこっち、きた。」と発言し、14分疑似オフライン化します。
本当にプロセスを落とすのではなく、Botが反応しない疑似オフライン状態にします。

通常モード復帰時に固定プリセットメッセージとして「まずは女子供から殺す」と発言します。
これは管理画面で編集しません。

設定項目:

* 通常反応トリガー
* 通常反応本文
* 疑似オフライン発生確率
* 疑似オフライン時間
* 疑似オフライン突入メッセージ
* 有効/無効

## 疑似オフライン

疑似オフラインは、Botプロセスを停止せず、一定時間反応しない状態として扱います。

疑似オフライン中の扱い:

* 通常反応しない
* モード抽選しない
* 自動投稿など時間駆動の処理は別途安全に扱う

通常モード復帰時には固定プリセットメッセージを投稿します。
