# いちよんロボ ver2.0 機能別設計

## 方針

ver2.0 では、機能をサーバーごとに管理します。
すべての運用データは基本的に `guild_id` を持ち、サーバーごとに分離します。

## 機能一覧

最初の機能一覧には以下を表示します。

* メンション反応
* 自動反応
* NGワード
* モード
* 自動投稿
* 特殊効果タグ

デッキ検索は最初の機能一覧には表示しません。
デッキ検索は「メンション反応 > 検索 > デッキ検索」に配置します。
破壊は独立機能として表示せず、特殊効果タグとして扱います。

## メンション反応

メンション反応は、Botへのメンションを起点に動く機能です。
「ランダム抽選」「検索」「限定機能」に分けます。

新規作成時は、まず種類を選ぶ想定です。

### ランダム抽選

ランダム抽選は管理画面から新規作成、編集、削除できます。
名言、おみくじ、お前も〇〇よな？はランダム抽選で実現します。

名言のメンション反応枠は削除不可です。

主な項目:

* キーワード
* 表示名
* 有効/無効
* 抽選候補

抽選候補の主な項目:

* 本文
* 画像
* 出やすさ
* 有効/無効
* 特殊効果タグ

同じサーバー内で既存と同じキーワードは保存不可です。
B と ABC のようなキーワードがある場合、ABC入力時はABCを優先します。
複数一致時は、長いキーワードを優先し、それでも同じなら作成が古いものを優先します。

### 検索

検索はシステム固定機能のみで、管理画面から新規追加できません。
デッキ検索は検索型で実現します。

デッキ検索の配置:

```txt
メンション反応
↓
検索
↓
デッキ検索
```

### 限定機能

限定機能は、特定ユーザーからBotへメンションされた時に特殊効果タグを必ず追加する機能です。

主な項目:

* 対象ユーザーID
* 表示名/メモ
* 付与する特殊効果タグ
* 有効/無効
* 説明

## 名言

名言は単独機能ではなく、メンション反応のランダム抽選枠として管理します。
文字、画像、文字+画像に対応します。

主な項目:

* 本文
* 画像
* 有効/無効

## おみくじ

おみくじは単独機能ではなく、メンション反応のランダム抽選枠として管理します。

主な項目:

* 結果名
* 本文
* 画像
* 出やすさ
* 有効/無効

管理画面では `weight` や `重み` ではなく「出やすさ」と表示します。

## 自動反応

サーバーごとに別管理します。

主な項目:

* トリガー
* 返答
* 画像
* 絵文字内部表記
* 一致方式
* 優先度
* 有効/無効
* 特殊効果タグ

既に自動反応には優先度を追加済みです。
複数トリガーが同時一致した場合は、優先度が高いものを優先し、同率ならランダムにします。

## NGワード

サーバーごとに別管理します。
NGワードを含む投稿には通常反応しません。

主な項目:

* NGワード
* 有効/無効

## 自動投稿

6/30自動投稿は単独機能にしません。
「自動投稿」という機能の中の1レコードとして扱います。

主な項目:

* 投稿名
* 本文
* 画像
* 投稿チャンネル
* 日付/スケジュール
* 繰り返し設定
* 有効/無効
* 投稿済み管理

6/30は以下のような設定値です。

```txt
名前: 6/30 サ終やめませんか？
日付: 毎年6月30日
本文: サ終やめませんか？
```

## 特殊効果タグ

メンション反応の抽選候補、自動反応、NGワードに付与する特殊効果タグを作成・編集する機能です。
メンション反応本体とモードには付与しません。

自動反応や抽選候補の編集画面では、作成済みの特殊効果タグを選択して付与するだけにします。
タグ自体の作成は別機能です。

特殊効果タグは危険度を持ちません。
タグ色を自由に設定できます。
管理者限定を持ちます。
効果設定はJSONB想定で柔軟にします。
追加投稿テキストは空白可です。

特殊効果タグ例:

* ライオ9倍
* さくらんぼ2回
* ミニいちよん
* しこっち特殊抽選
* 成田カウント加算

効果タイプ候補:

* 特殊行動確率倍率
* 次回行動回数追加
* カウント加算
* モード抽選
* 疑似オフライン抽選

### ミニいちよん

ミニいちよんは特殊効果タグとして扱います。
付与されたイベントが呼び出されるたびに抽選し、当選した場合だけ追加テキストを投稿します。
通常返答は先に実行します。

### しこっち抽選

しこっち抽選は特殊効果タグとして扱います。
1/444で当選したら `shikocchi_count` を1にします。

## デッキ検索

ver2.0の目玉機能の一つです。

これはBotにデッキを保存する機能ではありません。
Xから条件に合う最新ポストを検索して貼る機能です。
メンション反応の検索型として実現します。
最初の機能一覧には表示せず、「メンション反応 > 検索 > デッキ検索」に配置します。

コマンド例:

```txt
@いちよんロボ デッキ エルフ
@いちよんロボ デッキ エルフ ローテーション
```

処理イメージ:

```txt
Xから該当ポスト検索
↓
BEYOND到達っぽい投稿を候補にする
↓
添付画像にQRコードがあるか確認
↓
画像内のクラスアイコンが指定クラスか判定
↓
条件に合うポストを2件程度返す
```

管理画面で設定するもの:

* 使用可能チャンネル
* 返却件数
* フォーマット未指定時の扱い
* 許可外チャンネル時の案内

デッキ一覧CRUDは不要です。

## 検索/絞り込み

以下のデータは管理画面で検索/絞り込みできるようにします。

* メンション反応
* 抽選候補
* 自動反応
* NGワード
* 特殊効果タグ
* モード
* モード返答候補
* 自動投稿

Info表示は Infoボタン + モーダルで行います。
ホバー依存にはしません。
PC/スマホ両対応を前提にします。

## 破壊

破壊は独立機能ではなく、特殊効果タグとして扱います。
危険処理の実行は後続で、安全ガード込みで別途設計します。
