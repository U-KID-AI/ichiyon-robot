# リアクション絵文字対応

## 対象

- 自動反応
- メンション反応の抽選候補

DB backend の実行時、対象メッセージへリアクション絵文字を付ける。本文や画像がある場合は従来通り送信し、追加でリアクションを付ける。本文・画像がなくリアクション絵文字だけの場合は、Botメッセージを送らずリアクションだけ付ける。

## DB

自動反応は既存の `reactions.emoji_internal` を使う。

メンション反応の抽選候補は `mention_reaction_choices.emoji_internal` を追加する。

```sql
ALTER TABLE mention_reaction_choices
    ADD COLUMN IF NOT EXISTS emoji_internal TEXT;
```

## 管理画面

- 自動反応の一覧・作成・編集フォームに「リアクション絵文字」を表示
- メンション反応の抽選候補追加・編集フォームに「リアクション絵文字」を表示

通常絵文字とDiscordカスタム絵文字をそのまま入力する。

例:

- `🍒`
- `<:name:123456789012345678>`
- `<a:name:123456789012345678>`

## 実行時

リアクション追加に失敗してもBot全体は止めない。権限不足、未使用絵文字、不正形式などは警告ログだけ出して処理を続ける。

Bot招待権限には Add Reactions が必要。
